    Kit::Contract::BuiltInContracts

    This module contains some Built-in Contracts The easiest way to use them is to include `Kit::Contract::BuiltInContracts` in your class/module.
    Kit::Contract::BuiltInContracts::And

    Ensure all contracts are successful.
    Kit::Contract::BuiltInContracts::Any

    Always succeeds.
    Kit::Contract::BuiltInContracts::Args

    Allows to treat callable arguments as an array.
    Kit::Contract::BuiltInContracts::Array

    Enable Contracts on Array instances, and on elements at specific indeces.
    Kit::Contract::BuiltInContracts::ArrayHelper

    Array validation logic that does not need to live in the Class.
    Kit::Contract::BuiltInContracts::Delayed

    The actual contract is wrapped in a Callable and resolved on `call`.
    Kit::Contract::BuiltInContracts::Eq

    Ensure that the argument equals the saved Contract value.
    Kit::Contract::BuiltInContracts::Hash

    Enable Contracts on Hash instances, on keys themselves, on values themselves, and on elements at specific keys.
    Kit::Contract::BuiltInContracts::HashHelper

    Hash validation logic that does not need to live in the Class.
    Kit::Contract::BuiltInContracts::In

    Acts as an enum.
    Kit::Contract::BuiltInContracts::InstantiableContract

    Helper class to memoize Contract values.
    Kit::Contract::BuiltInContracts::IsA

    Ensure that the Contract value is of a given Type.
    Kit::Contract::BuiltInContracts::None

    Ensure no contract is successful.
    Kit::Contract::BuiltInContracts::NotEq

    Ensure that the argument is different from the Contract value.
    Kit::Contract::BuiltInContracts::Optional

    Ensure that Contract is obeyed when not nil.
    Kit::Contract::BuiltInContracts::Or

    Ensure at least one contract is successful.
    Kit::Contract::BuiltInContracts::RespondTo

    Ensure that the object `respond_to?` a specific method.
    Kit::Contract::BuiltInContracts::Tupple

    Less permissive version of Array.
    Kit::Contract::Engine

    The Engine is mainly used to handle all autoloading.
    Kit::Contract::Error

    When using contracts on method signatures (through `before`, `after`, `contract`) a `Kit::Contract::Error` exception is raised when a contract failure happens.
    Kit::Contract::Mixin

    Mixin that add `after`, `before`, `contract` class methods to enforce contracts on method signature.
    Kit::Contract::Services

    Namespace for Kit::Contract Services.
    Kit::Contract::Services::MethodWrapper

    Logic to rewrite methods in order to add signature contracts.
    Kit::Contract::Services::RubyHelpers

    Helpers methods to manipulate function signatures.
    Kit::Contract::Services::Runtime

    Namespace for runtime logic.
    Kit::Contract::Services::Store

    Local storage for registered Contracts.
    Kit::Contract::Services::Validation

    Namespace for Contract validation.
